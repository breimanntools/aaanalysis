# AGENTS.md

Vendor-neutral entry point for AI coding agents (Codex, Cursor, Gemini, Jules,
Aider, Cline, …). It mirrors the **load-bearing** subset of this repo's agent
instructions so any agent gets the same guardrails and orientation.

**AAanalysis** (Amino Acid analysis) is a Python framework for interpretable,
sequence-based protein prediction. Production/Stable on PyPI, BSD-3,
semver-strict from v1.x onward.

> **Authoritative sources.** `CLAUDE.md` (always-loaded core) and the
> path-scoped rules in `.claude/rules/` are the single source of truth and
> **override this file** on any conflict. Claude Code auto-loads them; other
> agents should read `CLAUDE.md` and the relevant `.claude/rules/*.md` for the
> file you are touching. This file is a stable pointer, not a second rulebook —
> keep detail there, not here.

## Hard rules (no exceptions, per-task approval)

1. **Never delete or move/rename a file without explicit user permission** —
   every mechanism (`rm`, `git rm`, `git mv`, `mv`, overwrites that erase
   content, `git clean/reset --hard/checkout --`, branch deletion). "Looks
   unused/stale/autogenerated" is not sufficient — ask first.
2. **Never push to a remote without explicit user permission** — `git push` in
   all forms, `gh pr create/merge`, `gh release create`. Local commits are fine;
   anything that leaves the local repo is not.

Approval is scoped to the exact action; it does not carry to the next one.

## Setup and feedback loops

```bash
uv pip install -e ".[dev]"                 # dev environment (.venv = py3.13)

pytest -m smoke -c tests/pytest.ini        # sub-minute smoke gate (load → CPP → TreeModel)
pytest tests -x -vv -c tests/pytest.ini    # full suite (always pass -c tests/pytest.ini)
pytest <path>::<Test>::<test> -x -vv -c tests/pytest.ini   # single test
cd docs && make html                       # build docs
```

Run tests with `-c tests/pytest.ini` (it carries `filterwarnings` and registers
the `regression`/`slow` markers). `aa.load_dataset(name=..., n=N)` returns `2N`
rows (n per class).

## Always-on guardrails (the highest-risk traps)

- **No `print(...)` in library code** — use `ut.print_out(...)`.
- **No imports from `aaanalysis._utils.*` outside `utils.py`** — go through
  `import aaanalysis.utils as ut`.
- **No cross-class backend imports** — a frontend imports backend helpers only
  from a shared `_backend/*.py` or its own dedicated `_backend/<subpkg>/`.
- **No bare `except Exception:`** to silence errors; no custom exception base —
  use bare `ValueError` / `RuntimeError`.
- **No ADR references in any `.py` file or in GitHub content** — state the
  rationale in plain language; ADR cross-refs stay in the decision layer only.
- **Every public parameter and return is type-annotated**; `Optional[T]` only
  when `None` is genuinely accepted.
- **Notebooks** (`examples/**`, `tutorials/**`): show DataFrames with
  `aa.display_df(df, n_rows=10, show_shape=True)`; end plot cells with
  `plt.tight_layout()` then `plt.show()`; commit executed outputs.
- **Reproducibility:** anything using randomness accepts a plumbed
  `random_state` / `seed` — never a bare, unseeded RNG.

## CONFIRM-FIRST surfaces (ask before editing)

`pyproject.toml`, `aaanalysis/__init__.py`, anything in `aaanalysis/_data/`,
`.github/workflows/*`, `aaanalysis/config.py`, `aaanalysis/template_classes.py`,
and any rename/removal of a symbol in `aaanalysis.__all__` (semver-relevant).

## Public API contract

The public API is exactly what `aaanalysis/__init__.py` re-exports; anything
starting with `_` is private and may change without deprecation. Each public
subpackage carries a front-door module docstring in its `__init__.py` with an
`__all__` synced to its re-exports. There is a second, opt-in convenience API,
`import aaanalysis.pipe as ap` (golden pipelines).
